Конфигурация гита

user.name (создается при регистрации на github)
user.email (почта с которой вы регистрировались)

git config --global user.name "Ivan Ivanou"
git config --global user.email "ivan_ivanou@mail.com"


Обычный порядок работы с гит

- создаем новую ветку:
    git checkout -b "your-branch-name"

      или

    (git branch "your-branch-name" и после этого git checkout "your-branch-name")

- делаем какую-то работу

- Команда git add применяется и для новых файлов и для измененных файлов.
    git add your_file.py

    или добавляем все файлы

    git add .

- коммитим изменения
    для конкретного файла
    git commit -m "your commit message" your_file.py

    или для всех файлов сразу

    git commit -m "your commit message"

- отправляем изменения на удаленный репозиторий

    git push

Исправляем ошибки

git checkout <commit_id> (ИД коммита можно посмотреть с помощью команды git log)


git checkout <file name> (возвращает файл в состояние последнего коммита)
Пример для git checkout <file name>: Мы сделали файл. Мы добавили его в гит. Мы сделали коммит - гит статус показывает
что коммитить нечего, в системе нет изменений которые мы не закоммитили. Мы вносим изменение в файл. НЕ КОММИТИМ,
а просто решаем отменить все внесенные в этот файл изменения. Вот для чего эта команда.


git reset HEAD <file name> (отменяет git add для файла. Содержимое файла не меняется)


git revert HEAD (отменяет последний коммит)
Пример: мы сделали изменения, сделали коммит ("wrong commit"), решили отменить.
Эта команда выведет тектовый редактор с примером сообщения для комита, отменяющего коммит.
(подтверждение сообщения и выход из редактора -- ввести :wq)

после этого последовательность коммитов будетвыглядеть как-то так
----"wrong commit"-------"revert wrong commit"---->
